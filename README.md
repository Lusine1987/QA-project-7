# QA-project-7
Учебный проект по тестированию API: Postman (коллекции, скрипты), REST API, JSON, сравнение реляционных и NoSQL БД.

### **Задание 1. REST API**

1. Самостоятельно изучи методы REST API, различия между методами, JSON, преимущества JSON имеет над XML и YAML, JSON Schema, сериализация и десериализация в контексте JSON. 
2. Выбери два публичных REST API. 
3. Изучи официальную документацию этих API.
4. Составь сравнительную таблицу для каждого API с указанием ссылок на выбранные REST API:

   - основные эндпоинты и поддерживаемые методы;
   - форматы запроса и ответа;
   - особенности авторизации и версии API;
   - отличия в подходах к дизайну и структуре.
5. В одном из выбранных публичных API найди пример JSON-запроса или ответа и добавь его в документ. Изучи структуру JSON: ключи, их типы данных. Запиши эти ключи и типы в отдельный список. Укажи ссылку на анализируемый REST API. 
6. Выполненное задание оформи в документе `task_1.md`.

### **Задание 2. Postman**

1. Установи Postman, зарегистрируй свой аккаунт (так будет удобнее собирать проекты).
2. Изучи в Postman следующие функциональности:
   - настройка окружения;
   - создание коллекций
   - создание запросов в коллекциях;
   - добавление сниппетов;
   - запуск автотестов для коллекции.
3. Для одного из выбранных в предыдущем задании публичных API примени полученные навыки работы с Postman: настрой рабочее окружение, создай коллекцию запросов (минимум 1), добавь разные запросы к API (в коллекции минимум 3 запроса, разные по методам), добавь к запросам сниппеты, выполни запросы и проанализируй полученные ответы.
4. Экспортируй получившуюся библиотеку запросов в файл `.json`. Убедись, что коллекция может быть импортирована и использована другим пользователем без дополнительных настроек (правильно настроенные переменные, параметры).
5. Экспортированный файл с расширением `.json` загрузи в файловый сервис.

### **Задание 3. Базы данных**

1. Самостоятельно изучи отличия реляционных БД от нереляционных, а также плюсы и минусы каждого вида, отличия СУБД одного типа (например, чем отличается реляционная MySQL от реляционного PostgreSQL).
1. Выбери один из популярных онлайн-сервисов, которым ты пользуешься (социальная сеть, мессенджер, интернет-банк и т.д.)
3. Сформулируй и зафиксируй гипотезу о том, какой тип БД (реляционная, нереляционная или гибридный подход) является основным для выбранного сервиса? Аргументируй свой ответ, опираясь на функциональность сервиса *(не используй на этом этапе интернет, суть задания научиться формулировать гипотезы, основываясь на информации и своих знаниях, что входит в рабочие задачи тестировщика)*.
4. Разбей сервис на 3-4 ключевые функции. Для каждой функции опиши: какие данные она использует, и какой тип БД для них, по твоему мнению, больше подходит и почему.
5. Опиши 2-3 идеи для тест-кейсов для проверки выбранных типов БД.
6. Выполненное задание оформи в документе `task_3.md`.
